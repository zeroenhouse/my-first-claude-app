# Claude Code - 組織構造設定

このワークスペースは、複数の役割を持つ仮想組織として設計されています。

## 📁 ディレクトリ構成

```
/workspaces/my-first-claude-app/
├── データ/                    # 共有コンテキスト・入力情報
├── CMO（マーケティング責任者）/ # マーケティング・市場調査
├── CPO（プロダクト責任者）/   # プロダクト開発・設計
├── CFO（財務責任者）/         # 財務・請求書・契約書
├── CSO（営業責任者）/         # 営業・顧客対話
└── CTO（技術責任者）/         # 技術・プロンプト・インフラ
```

## 👥 各ロールの責務

### 📊 CMO（マーケティング責任者）
- 市場調査と競合分析
- マーケティング戦略の立案
- SNS投稿、ブログ、YouTube動画脚本の作成
- キャンペーン企画

**関連フォルダ**: `/CMO（マーケティング責任者）/`

### 🛠 CPO（プロダクト責任者）
- プロダクト企画・設計
- 機能定義・仕様書作成
- ロードマップ管理
- ユーザーフィードバック分析

**関連フォルダ**: `/CPO（プロダクト責任者）/`

### 💰 CFO（財務責任者）
- 予算管理・財務計画
- 請求書・領収書発行
- 契約書作成・管理
- 経費報告書

**関連フォルダ**: `/CFO（財務責任者）/`

### 🤝 CSO（営業責任者）
- 顧客との対話記録
- セールスコピー作成
- 商談管理
- 顧客フィードバック収集

**関連フォルダ**: `/CSO（営業責任者）/`

### 💻 CTO（技術責任者）
- 技術スタック選定
- コード設計・実装
- プロンプト・指示の最適化
- インフラ・デプロイメント

**関連フォルダ**: `/CTO（技術責任者）/`

### 📚 データ（コンテキスト）
- プロジェクト全体の日記・考え事
- クライアント資料・PDF
- 一時情報・リサーチ結果
- すべてのロールが参照するベースデータ

**関連フォルダ**: `/データ/`

## 🎯 使用方法

1. **入力**: `データ/` フォルダに日記、資料、クライアント情報を格納
2. **処理**: 各ロール別フォルダで担当業務を実行
3. **出力**: 各ロールがアウトプット（提案書、契約書、動画脚本など）を生成

## ✨ 設計のポイント

- **整理専門**: ファイルの配置と構造化は人間が担当
- **AI実行**: 各ロールの仕事はAIが対応（人間以上のパフォーマンス）
- **漏れなくダブりなく**: 各ロールの責務を明確に分離
- **相互連携**: CSO→CMO→CPOへの営業情報フロー など、ロール間の協力を可能に
