# 📋 CLO統合ドキュメント - GiGO我孫子プロジェクト 法務計画

## 📌 CLO（Chief Legal Officer）の位置付け

**GiGO我孫子クレーンゲーム事業** は、以下の複数の法定分野を跨ぐプロジェクト：

```
┌─────────────────────────────────────────┐
│  買収契約法務  ←→  不動産契約         │
│         ↓                    ↓           │
│  景品規制法   ←→  風営法適用          │
│         ↓                    ↓           │
│  労務管理法   ←→  消費者保護法        │
│         ↓                    ↓           │
│  個人情報保護  ←→  税務申告            │
└─────────────────────────────────────────┘
```

**CLO の役割**: これら全 8 つの法定分野を統合管理し、  
リスクを最小化しながら、ビジネス目標を達成させる

## ⚖️ GiGO我孫子フォーカスの 8 つの法的リスク

### 1. 「景品表示法」による景品規制 🔴 CRITICAL

**現在の計画**: 460台にクレーンゲーム配置

**景表法の規制**:
```
新聞広告や SNS で「商品が当たる」とうたう場合
├ 景品の種類・個数に上限がある
├ 「高確率で取得可能」と表示すると景表法違反
└ 消費者庁から「行政処分」＆「罰金」対象

【例】
❌ NG: 「今日のこの機種は82%の確率で取得！」
✅ OK: 「クレーンゲーム機です。力試しにチャレンジください」
```

**CLO 対応タスク** (4月完了):
```
□ 消費者庁「景品表示法ガイドライン」を読込
□ 各景品カテゴリ（食品・フィギュア・日用品）の法定額を確認
□ 460台の「立て看板・機械表示」が合法か確認
□ 景品メーカー（OTAICHI 参考、他）の事例を研究
□ 外部：消費者庁の地方ブロック事務所に相談
```

### 2. 「風営法」との関係 🟡 MEDIUM

**日本の法律上**: クレーンゲーム = 風営法の規制外

```
【パチンコ・スロット】 🚫 風営法対象
├ 出玉の換金性あり
└ 遊技機として厳密に規制

【クレーンゲーム】 ✅ 風営法対象外
├ 景品は玩具・食品など（金銭性なし）
└ より自由な営業が可能
```

**ただし千葉県や市区町村の条例確認が必須**:

**CLO 対応タスク** (4月完了):
```
□ 千葉県警察本部 - 風営課に電話相談
  └ 「クレーンゲーム専門店」として営業許可の必要性確認
□ 我孫子警察署に営業予定届出（慣例）
□ 地元・我孫子市庁舎の産業課に確認
□ スマイルフィールド管理会社経由の確認も実施
```

### 3. 「買収契約法務」- M&A契約 🔴 CRITICAL

**現在の計画**: GENDA GiGO Entertainment から営業権買収（5,000万円）

```
【典型的なリスク】
├ 隠れた債務（未払い金・買戻し義務）
├ 契約に紐づく義務の引き継ぎ
├ 従業員の雇用義務
├ 顧客データ・営業秘密の取扱い
└ クローイング条件（融資承認など）
```

**CLO 対応タスク** (4月開始～5月完了):
```
【Phase 1: 買収前の調べ（DD - デューデリジェンス）】
□ GENDA GiGO に情報開示請求
  ├ 過去3年の月商推移
  ├ 現在の賃借契約書・管理費の詳細
  ├ 在庫・固定資産のリスト
  ├ スタッフ1人の雇用契約書
  └ 訴訟・行政処分の有無
□ 弁護士と一緒に情報を法務チェック

【Phase 2: 契約書作成】
□ 弁護士と共同で買収契約書ドラフト
  ├ Purchase Price の分割払い条件
  ├ Closing condition（クロージング条件）
  ├ 売主の表明・保証（warranty）
  ├ 買主の表明・保証
  ├ 調査期間（30～45日）
  └ Indemnification（損害賠償条項）
```

### 4. 「不動産賃貸借契約」法務 🔴 CRITICAL

**現在の仮定**: 「スマイルフィールド我孫子」内に入居

```
【現在の状況】
VZONE（パチンコ）と同一施設
├ 建物オーナー ← スマイルフィールド管理会社
├ VZONE テナント
└ GiGO（買収対象） テナント

【買収後】
新しい運営者 ← 回遊館グループ
├ 直接スマイルフィールトと賃貸借契約
└ VZONE との隣接関係を維持
```

**CLO 対応タスク** (4月～5月):
```
【確認項目】
□ 現GiGO賃貸借契約書を入手
  ├ 月賃料（現在××万円？）
  ├ 契約期間（×年）
  ├ 原状回復義務（転出時にどこまで直すか）
  ├ 更新条件（何年ごと？）
  └ 解約条項（中途解約時の違約金）

□ スマイルフィールド管理会社と事前協議
  ├ 「新運営者への承継が可能か」確認
  ├ 賃料値上げはあるか（現在比+50%?)
  ├ 敷金・保証金の追加徴収はあるか
  ├ 改装工事を実施する場合の許可範囲
  └ VZONEとの「共有部分」での紛争回避策

□ 新規賃貸借契約書の作成
  ├ 契約期間：推奨 10年以上（初期投資回収のため）
  ├ 賃料：固定 or 2年ごと改定？
  ├ 敷金：月賃料の 6ヶ月分
  └ 原状回復範囲：「通常の営業損耗」としない（カスタマイズ多い）
```

### 5. 「労務管理法」遵守 🟡 MEDIUM

**現在の計画**: スタッフ 29 名、月給 750 万円

```
【法定要件】
├ 最低賃金の遵守（千葉県 1,150 円/時）
├ 労働契約書の作成・交付（各スタッフに）
├ 雇用保険・労災保険への加入
├ 給与明細書の発行（毎月）
├ 36 協定の届出（残業時間の上限設定）
└ 就業規則の周知（10 人以上の事業所は必須）
```

**CLO 対応タスク** (5月～6月):
```
【雇用前】
□ 就業規則の作成
  ├ 勤務時間（営業中 10:00～24:00、3シフト）
  ├ 給与（月給制 or 時給制？）
  ├ 休日（週何日？）
  ├ 残業手当（時給×1.25 倍以上）
  ├ 福利厚生（有給休暇、社会保険）
  ├ 懲罰規定（無断欠勤時など）
  └ 秘密保持・個人情報取扱い
□ 労働契約書テンプレートの作成
  ├ title（正社員/アルバイト）
  ├ 給与（月額××円 or 時給××円）
  ├ 試用期間（3ヶ月など）
  ├ 有給休暇（10日/年など）
  └ 契約終了条件

【雇用時】
□ 各スタッフとの労働契約書署名
□ 就業規則の配布・説明
□ 雇用保険・労災保険の加入手続き
□ 社会保険（厚生年金・健康保険）加入

【運営中】
□ 毎月の給与計算・支払い
  ├ 基本給×人数
  ├ 残業手当（超過分）
  ├ 社会保険料控除
  ├ 所得税等控除
  └ 給与明細書の発行
□ 年末調整
□ 社会保険料とまとめての納付
```

### 6. 「消費者保護法」対応 🟡 MEDIUM

**リスク例**:
```
顧客 A: 「爪の強度が強すぎて取れない。詐欺だ！」
        → Twitter/「Google Map」で低評価投稿
        → 法的クレームに発展する可能性

顧客 B: 「景品をもらったら中身が壊れてた」
        → 景品原価が 300 円なのに交換・返金要求
        → 「消費者契約法」違反ではないか？
```

**CLO 対応タスク** (5月～6月):
```
□ 「景品交換規程」の作成
  ├ 「新品・未検査」の明示
  ├ 「取得後の検査は自己責任」の免責文言
  ├ 不良品の場合の交換手続き（返金不可）
  └ 交換品がない場合はコイン分返却（←金銭返金NG）

□ 店内掲示板・HP での規定表示
  ├ 「機械の不具合時の対応」
  ├ 「景品交換手続き」
  ├ 「苦情申立て窓口」
  └ 「景表法・風営法」への合法性表明（簡潔に）

□ クレーム対応マニュアルの作成
  ├ 「機械が動かない」→ 即座に返金 or 代替機
  ├ 「景品に不具合」→ 無料交換（同等品に限定）
  ├ 「詐欺だ」との抗議 → 記録・報告・弁護士相談
  └ Google Map など SNS での低評価への対応
```

### 7. 「個人情報保護」対応 🟡 MEDIUM

**取得する情報**:
```
├ LINE 公式アカウント登録 → 電話番号・年齢
├ 会員カード → 氏名・住所・電話
├ カード決済 → カード番号（PCI-DSS 準拠）
└ 来店客のカメラ映像 → 防犯以外は法的グレーゾーン
```

**CLO 対応タスク** (5月～6月):
```
□ 「個人情報保護ポリシー」の作成
  ├ 取得する情報の種類
  ├ 用途（マーケティング・セール通知など）
  ├ 第三者提供の可否（OTAICHI本社への共有など）
  ├ 保管期間（退会後いつまで保管？）
  ├ 削除請求への対応
  └ 個人情報侵害時の対応手順

□ LINE・SNS の利用規約確認
  ├ 「セール通知」同意の取得フロー
  ├ 「配信停止」への迅速な対応
  └ 月〇回以上の配信時の事前通知

□ POS システムのセキュリティ確認
  ├ ベンダーの「個人情報管理体制」確認
  ├ 暗号化・バックアップの実施状況
  ├ 定期的なセキュリティ監査
  └ 従業員のアクセス権限制限

□ スタッフの秘密保持契約
  ├ 顧客情報の漏洩禁止
  ├ SNS での社内情報投稿禁止
  └ 退職時の情報返却確認
```

### 8. 「税務申告・会計」法務 🟡 MEDIUM

**必須事項**:
```
【設立時】
├ 法務局：法人設立登記
├ 税務署：法人設立届出書
├ 千葉県：法人設立報告書
└ 我孫子市：事業開始届出書

【毎年】
├ 法人所得税申告書（確定申告）
├ 消費税申告書（売上 > 1,000 万円の場合）
├ 地方法人税・事業税申告書
└ 給与支払い報告書（従業員 29 人分）

【毎月】
├ 売上データの記録（機器別・時間帯別）
├ 給与計算・社会保険料納付
└ 経費の領収書保管
```

**CLO 対応タスク** (4月～7月):
```
□ 法人設立形態の決定
  ├ A) 新設法人（GiGO我孫子店㈱など） → 新・独立採算
  └ B) 既存法人の支店化 → 親会社と共枠

□ 税理士・会計士の選定・契約
  ├ 飲食・興業業種の経験者
  ├ 中小企業向け対応可能
  └ 月次報告・決算支援体制

□ 会計・売上管理システムの整備
  ├ 日々の売上データの正確な記録
  ├ 経費の領収書管理
  ├ 給与計算システムの導入
  └ 適格請求書発行事業者の登録（課税事業者の場合）

□ 固定資産台帳の作成
  ├ 460 台のクレーン機の購入価格・納入日
  ├ 5 年償却で月額∼万円
  ├ 改装工事の耐用年数対応
  └ 売却・処分時の処理フロー
```
