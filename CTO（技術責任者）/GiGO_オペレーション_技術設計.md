# CTO - GiGO我孫子 オペレーション・技術設計

## 🔧 CTO の責務（このプロジェクト）

クレーンゲーム専門店として、毎日安定した営業を実現するための  
**技術・システム・オペレーション** を設計する。

## 💰 決済・売上管理システム

### マルチシステム構成

```
【顧客】
  ↓ 選択
  ├→ 現金投入 → 両替機 → コイン配出
  ├→ カード → IC読み込み機 → 残高確認
  └→ QR決済 → スマホ → クラウド連携
  ↓
【クレーン機器】
  ├ 使用状況をリアルタイム記録
  ├ 売上データ送信
  └ 機械トラブルを自動検知
  ↓
【POS / 売上管理システム】
  ├ 時間別・機器別・景品ジャンル別の売上集計
  ├ 在庫管理（景品の入出庫）
  ├ スタッフシフト・給与管理
  └ リアルタイムダッシュボード表示

【クラウド（本部）】
  ├ 遠隔監視
  ├ 異常検知アラート
  └ リモート設定変更
```

### 期待効果
- **リアルタイム売上把握**: 日単位・時間単位での営業状況監視
- **異常検知**: 機械故障を即座に検知 → スタッフ呼び出し
- **データ分析**: 「どの機種がよく使われるか」を分析 → 次の景品購入に活かす

## 🧹 日常オペレーション・清掃計画

### Daily Checklist（毎日実施）

#### 営業前（09:00〜10:00）
- [ ] 全機器の電源ON・起動確認
- [ ] フロア全体の清掃（床・景品台）
- [ ] 両替機の紙幣補充
- [ ] POS/売上システムの起動確認
- [ ] スタッフの身だしなみ確認

#### 営業中（営業時間中）
- [ ] 1時間ごとにフロアの清掃 & 景品補充
- [ ] 故障報告があったら即座に対応
- [ ] 顧客からのクレーム記録
- [ ] 定時（毎正時）に売上確認

#### 営業終了時（23:00〜24:00）
- [ ] 最終清掃・整理整頓
- [ ] 両替機の終日集計・金庫保管
- [ ] 明日の備品チェック
- [ ] スタッフの連絡帳記載（引継ぎ事項）

## 🚨 トラブル対応フロー

### シナリオ1: 機器が動かない
```
顧客報告
   ↓
フロアスタッフが機器確認
   ↓
電源/コイン詰まり？ → 簡易対応 (60秒以内)
   ↓
改善なし → メンテナンス呼び出し
   ↓
修理対応 (15分以内)
   ↓
修理完了 → 顧客に謝罪 + 代替機案内
```

### シナリオ2: 顧客が景品を取れない
```
顧客が難しいと申告
   ↓
スタッフが実演で「こうやってください」
   ↓
顧客が再チャレンジ
   ↓
取れた場合 → 「取れました！おめでとう」
   ↓
取れない場合 → 「難しいですね。別の機種はいかがですか」
             + 別機種のコツ紹介
```

### シナリオ3: クレーム対応
```
顧客クレーム受付
   ↓
副店長がヒアリング（誠実に聞く）
   ↓
原因を特定 & 謝罪
   ↓
改善策を提示
   ├ 技術的問題 → 機械調整
   ├ スタッフ対応 → 教育実施
   └ システム問題 → 本部連携
```
