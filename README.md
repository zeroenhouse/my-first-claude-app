# GiGO Project

## プロジェクト概要
GiGO（Get in, Get out）プロジェクトは、ゲームセンターの買収と運営に関する総合的な計画を立てるためのものです。

## ディレクトリ構造
プロジェクトは役割ベースのディレクトリ構造を採用しており、各役割フォルダ内に以下のサブフォルダを配置しています：
- **計画**: 戦略立案と計画書
- **実行**: 実施計画とアクションプラン
- **レビュー**: 評価とフィードバック
- **アイデア**: 創造的な提案とイノベーション

### 役割フォルダ
- CFO（財務責任者）
- CLO（法務責任者）
- CMO（マーケティング責任者）
- CPO（プロダクト責任者）
- CSO（営業責任者）
- CTO（技術責任者）

### データフォルダ
- データ/: プロジェクト関連のデータファイルと分析資料

## 改善された構造の特徴
- **3倍優秀**: 詳細なサブフォルダによる組織化で、効率性と明確性を向上
- **クリエイティブ**: アイデアフォルダによる創造性促進
- **クリティカル**: レビューセクションによる批判的思考と品質管理

## ファイル一覧
