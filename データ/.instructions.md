# データ（コンテキスト）フォルダの役割

## 📚 責務

このフォルダは**全ロール共通の情報・コンテキスト**を格納します。

### 保管するものの例

1. **プロジェクト全体の情報**
   - プロジェクト日記・方針メモ
   - 意思決定ログ
   - 一時的なアイディア帳

2. **クライアント・顧客情報**
   - クライアント企業プロフィール
   - PDFドキュメント・資料
   - 提供されたデータ・ファイル

3. **市場・業界情報**
   - 市場レポート
   - 競合分析資料
   - トレンド情報

4. **その他の共有情報**
   - 外部ツール・APIドキュメント
   - ライセンス情報
   - 参考リンク集

## 📁 推奨フォルダ構成

```
データ/
├── 日記/
│   └── [日付]_[タイトル].md
├── クライアント資料/
│   ├── [クライアント名]/
│   │   ├── 企業情報.md
│   │   ├── 提供資料/
│   │   └── 要件定義.md
│   └── [プロジェクト名]/
├── 市場データ/
│   ├── 業界レポート/
│   ├── 競合分析/
│   └── トレンド情報/
├── 外部資料/
│   ├── API仕様/
│   ├── ドキュメント/
│   └── 参考リンク.md
└── その他/
    ├── ライセンス/
    └── チュートリアル/
```

## ⚠️ 重要なポイント

- **全ロールがアクセス**: CMOもCPOもCFOも、このフォルダから必要な情報を参照
- **整理が命**: ファイルの配置が不適切だと、各ロールが情報を見つけられない
- **人間の責務**: ファイルの整理・分類は人間が担当（AIは処理担当）
- **重複を避ける**: 同じ情報を複数フォルダに置かず、ここから参照する仕組みに

## 🔗 各ロールとの連携

- **CMO** ← `市場データ/`、`クライアント資料/` から市場情報を取得
- **CPO** ← `クライアント資料/`、`市場データ/` から要件・ニーズを取得
- **CFO** ← `クライアント資料/` から契約情報を取得
- **CSO** ← `クライアント資料/` から顧客情報を取得
- **CTO** ← `外部資料/` から技術情報を取得
